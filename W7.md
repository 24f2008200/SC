## Full Explanation

---

### Line 1 — `#!/bin/bash`

Called a **shebang**. Tells the OS which interpreter to use when running the script.

```
#!        →  "use this interpreter"
/bin/bash →  path to the Bash shell
```

Without it, the OS might run the script with `sh` (a simpler shell) which may not support all bash features. Always the **first line** of a bash script.

---

### Line 2 — `grep -oP "^[^,]+(?=,[0-9]*[05]$)"`

#### The Flags

| Flag | Meaning |
|---|---|
| `-o` | Print **only the matched portion**, not the whole line |
| `-P` | Use **Perl-Compatible Regex (PCRE)** — needed for lookahead `(?=...)` |

---

#### The Regex — Broken Down Piece by Piece

```
^[^,]+(?=,[0-9]*[05]$)
```

| Part | Meaning | Example Match |
|---|---|---|
| `^` | Start of line | — |
| `[^,]+` | One or more characters that are **not a comma** | `John Doe` |
| `(?= ... )` | **Lookahead** — what must follow, but is **not captured** | — |
| `,` | A literal comma inside the lookahead | `,` |
| `[0-9]*` | Zero or more digits | `3`, `4`, `` |
| `[05]` | Must end in `0` or `5` | `0` in `30` |
| `$` | End of line | — |

---

#### Walking Through Each Row

```
Name,Age          →  [^,]+ matches "Name"
                     lookahead checks ",Age" → "Age" doesn't end in 0/5 ❌ NO MATCH

John Doe,30       →  [^,]+ matches "John Doe"
                     lookahead checks ",30"  → ends in 0 ✅ PRINTS "John Doe"

Jane Smith,22     →  [^,]+ matches "Jane Smith"
                     lookahead checks ",22"  → ends in 2 ❌ NO MATCH

Alice Johnson,25  →  [^,]+ matches "Alice Johnson"
                     lookahead checks ",25"  → ends in 5 ✅ PRINTS "Alice Johnson"

Bob Brown,45      →  [^,]+ matches "Bob Brown"
                     lookahead checks ",45"  → ends in 5 ✅ PRINTS "Bob Brown"
```

---

#### Why Lookahead `(?=...)` is the Key

Without `-o`, grep prints the **whole line**. With `-o` it prints only the match.

The lookahead lets you say:
> *"Match the name, BUT ONLY IF it is followed by a qualifying age — and don't include the age in the output"*

```
"John Doe,30"
      ↑         ↑
   matched   lookahead checks this
   printed   NOT printed
```

This is why the output is **names only** with no commas or ages.

---

### Full Flow Summary

```
stdin (CSV)
    │
    ▼
grep reads each line
    │
    ├─ Does the name part exist?        [^,]+
    ├─ Is it followed by a comma?       (?=,
    ├─ Then any digits?                 [0-9]*
    └─ Ending in 0 or 5?               [05]$)
         │
         ✅ YES → print just the name (-o)
         ❌ NO  → skip the line
```

## `(?>` and `(?<` in Regex (PCRE)

---

### `(?<` — Lookbehind

Checks what comes **before** the match. Mirror of lookahead `(?=)`.

#### Two Types

| Syntax | Name | Meaning |
|---|---|---|
| `(?<=...)` | Positive lookbehind | Match only if **preceded by** this |
| `(?<!...)` | Negative lookbehind | Match only if **NOT preceded by** this |

#### Examples

```bash
# Positive lookbehind — extract age only if preceded by a comma
echo "John,30" | grep -oP "(?<=,)[0-9]+"
# output: 30

# Negative lookbehind — match digits NOT preceded by comma
echo "John,30" | grep -oP "(?<!,)[0-9]+"
# output: nothing (30 IS preceded by comma)
```

---

### Lookahead vs Lookbehind — Side by Side

```
(?=  ...)   Lookahead         →  what comes AFTER  the match
(?<= ...)   Lookbehind        →  what comes BEFORE the match
(?!  ...)   Negative lookahead  →  must NOT follow
(?<! ...)   Negative lookbehind →  must NOT precede
```

#### Visual

```
(?<=,)[0-9]+
      ↑
"look BACK, confirm comma is there"
"then match the digits"


^[^,]+(?=,[0-9]*[05]$)
             ↑
      "look FORWARD, confirm age ends in 0/5"
      "then match the name"
```

---

### `(?>` — Atomic Group

Tells regex: **"once matched, never give it back"** — disables backtracking inside the group.

```bash
# Normal group — allows backtracking
grep -P "(a+)b"       "aaab"   # ✅ matches

# Atomic group — no backtracking
grep -P "(?>a+)b"     "aaab"   # ❌ fails — grabs all a's, won't give any back for b
```

#### Why Use It?

**Performance** — prevents catastrophic backtracking on large inputs.

```bash
# Without atomic — regex engine tries every combination
grep -P "([0-9]+)*X"

# With atomic — commits immediately, much faster
grep -P "(?>[0-9]+)*X"
```

---

### All Four Together — Cheat Sheet

| Syntax | Type | Question it answers |
|---|---|---|
| `(?=abc)` | Lookahead | Is `abc` **after** my match? |
| `(?!abc)` | Neg. Lookahead | Is `abc` **NOT after** my match? |
| `(?<=abc)` | Lookbehind | Is `abc` **before** my match? |
| `(?<!abc)` | Neg. Lookbehind | Is `abc` **NOT before** my match? |
| `(?>abc)` | Atomic Group | Match `abc`, **never backtrack** into it |

---

### Key Rule

> All of these are **non-capturing** — they check conditions but never appear in the output. Only available with `-P` (PCRE) in grep.
